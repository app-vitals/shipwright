## Method note

This is the third `/test-migration` pass (prior: 2026-07-16, committed as this file's
history shows). Rather than re-read all ~241 test files cold, this pass diffed the
current file tree against the prior run's findings and re-verified each open item
directly, then did a fresh full-suite discovery pass to catch anything the diff approach
would miss:

1. Enumerated every test file via the Phase 2 framework matrix's glob patterns (`bun
   test`'s default discovery for `*.unit.test.ts` / `*.integration.test.ts` /
   `*.smoke.test.ts` / `*.content.test.ts`, plus `site/**/*.spec.ts`, plus
   `{admin,metrics}/e2e/**/*.e2e.ts`).
2. Re-checked every open item from the 2026-07-16 report (6 net-new tests, 2 promote/rename
   fixes, 2 net-new infra items) against the current tree by direct file-existence and
   content checks — not assumed from memory.
3. Cross-referenced Phase 1's "new since last inventory" file list (~17 new source files)
   against the test-file glob to find newly-introduced coverage gaps.
4. Re-ran the isolation-contract grep sweep (`mock.module(`, `global.fetch =`,
   `global.console =`, `DATABASE_URL` inside `*.unit.test.ts`) across the full current
   suite, not just the delta, to catch any regression introduced by unrelated PRs since
   the last pass.
5. Confirmed `.github/pull_request_template.md` and `bunfig.toml`'s `pathIgnorePatterns`
   directly rather than trusting the prior doc's snapshot.

**Speed measurement: not re-run this pass**, same reasoning as the prior two passes — no
local Postgres in this sandbox (`DATABASE_URL_SHIPWRIGHT_*` unset), so a `bun test` run here
would silently skip every DB-backed integration test and produce misleading timings.
Static review (line counts, import/mock shape) was used instead; no evidence found of any
test breaching a hard cap. **Note (see Speed measurement section below for the full
correction): a real per-layer measurement already exists at `docs/test-readiness/speed-baseline.md`
on `main` since 2026-07-15 — none of the three `/test-migration` passes to date checked
for it, so "not run" below describes this pass's own static-review method, not the
repo's actual measurement state.** See `project_shipwright_metrics_full_suite_flake` and
`project_shipwright_admin_full_suite_flake` in memory for the two known pre-existing
full-suite flakes (unrelated to layer/speed, not relevant to this audit).
