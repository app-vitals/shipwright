## Method note

This is the fifth `/test-migration` pass (prior: 2026-07-16, 2026-07-19, 2026-07-21,
committed as this file's history shows). Rather than re-read all ~249 test files cold,
this pass diffed the current file tree against the 2026-07-21 pass's findings and
re-verified each open item directly, then did a fresh full-suite discovery pass to catch
anything the diff approach would miss:

1. Enumerated every test file via the Phase 2 framework matrix's glob patterns (`bun
   test`'s default discovery for `*.unit.test.ts` / `*.integration.test.ts` /
   `*.smoke.test.ts` / `*.content.test.ts`, plus `site/**/*.spec.ts`, plus
   `{admin,metrics}/e2e/**/*.e2e.ts`).
2. Re-checked every open item from the 2026-07-21 report (`admin/src/clock.ts` net-new,
   low-priority/optional; `mcp-server/src/main.ts` net-new, real gap) against the current
   tree by direct file-existence checks — not assumed from memory. Both remain untested;
   see Net-new section.
3. Cross-referenced Phase 1's "new since the 2026-07-21 inventory" file list (3 new
   non-test source files: `admin/src/agent-members.ts`, `lib/agent-default-tools.ts`,
   `site/src/lib/html-escape.ts`) against the test-file glob to find newly-introduced
   coverage gaps.
4. Re-ran the isolation-contract grep sweep (`mock.module(`, `global.fetch =`,
   `global.console =`, `DATABASE_URL` inside `*.unit.test.ts`) across the full current
   suite, not just the delta, to catch any regression introduced by unrelated PRs since
   the last pass. Counts are byte-for-byte unchanged from the 2026-07-21 pass (36 / 0 / 0
   / 3, all previously-confirmed non-violations) — no regression.
5. Discovered and corrected a stale count in the 2026-07-21 doc: it stated 12 `site/`
   Playwright specs, but `git ls-tree` at that doc's own baseline commit (`82139c09`)
   shows 17 already existed then — the "12" was an inherited undercount, not a real
   figure. This pass's own count (20) is directly `git`/`find`-verified against the
   working tree, not carried forward.
6. Confirmed the 3 new non-test source files' test status directly: `admin/src/agent-members.ts`
   ships with `admin/src/agent-members.unit.test.ts` in the same commit (matches the
   repo's "tests land with the code" convention); `lib/agent-default-tools.ts` and
   `site/src/lib/html-escape.ts` have no test file yet (both already flagged in Phase 1/2
   as low-risk, non-blocking gaps).

**Speed measurement: not re-run this pass**, same reasoning as all three prior passes — no
local Postgres in this sandbox (`DATABASE_URL_SHIPWRIGHT_*` unset), so a `bun test` run here
would silently skip every DB-backed integration test and produce misleading timings.
Static review (line counts, import/mock shape) was used instead; no evidence found of any
test breaching a hard cap. A real per-layer measurement continues to exist at
`docs/test-readiness/speed-baseline.md` on `main` (unchanged since 2026-07-15, already
cross-referenced into `test-system.md`'s Speed budgets section since the 2026-07-19 pass
per T-067/PR #1930 — no further correction needed this cycle). See
`project_shipwright_metrics_full_suite_flake` and `project_shipwright_admin_full_suite_flake`
in memory for the two known pre-existing full-suite flakes (unrelated to layer/speed, not
relevant to this audit).
